# Description

!!! note

    The below checks require `manifest.json` to be present.

These checks ensure your dbt models are properly documented. Use them to verify that every model has a populated description, that descriptions match a required regex pattern, that documentation coverage meets a minimum threshold, and that docs blocks live in the same directory as the model they describe. Well-documented models reduce onboarding time and make your dbt project easier to navigate.

::: manifest.models.description
