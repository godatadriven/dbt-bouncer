# Directories

!!! note

    The below checks require `manifest.json` to be present.

These checks enforce directory and file structure conventions for your dbt models. Use them to verify that models reside in the expected directories, that SQL file names match a required pattern, that YAML property files are co-located with their models, and that schema names follow naming conventions. A consistent project layout makes it easier for team members to find, review, and maintain models as the project scales.

::: manifest.models.directories
