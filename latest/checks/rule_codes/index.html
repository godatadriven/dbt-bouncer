Rule codes#

Every built-in check has a unique rule code: a two-letter prefix identifying the group of resources it applies to, followed by a three-digit number. Codes are stable — once published, a code always refers to the same check.

Referencing a check by its code#

Anywhere a check is named you may use its code instead. These two config entries are equivalent:

manifest_checks:
  - name: check_model_description_populated
  - code: MO021

Codes are also accepted in the skip_checks meta key on a dbt resource:

models:
  - name: my_model
    meta:
      skip_checks:
        - MO021

To browse codes from the command line, use dbt-bouncer list, optionally filtered with --group:

dbt-bouncer list --group manifest_checks
